Sistem Monitoring Suhu Data Center Berbasis IoT

Laporan Proyek: Data Center Monitor (IoT)

Disusun oleh: Wildan Ulul Aufa

Latar Belakang Proyek

Tantangan: Kebutuhan akan pemantauan suhu dan kelembaban yang konsisten di data center untuk mencegah kerusakan hardware dan downtime server.

Solusi: Proyek ini menghadirkan sebuah sistem IoT yang efektif dan efisien untuk memantau kondisi lingkungan secara real-time. Sistem ini mengandalkan server lokal yang terintegrasi dengan perangkat keras (ESP8266) dan aplikasi mobile, memberikan kontrol penuh tanpa ketergantungan pada layanan cloud eksternal.

Tujuan Proyek

  • Pemantauan Otomatis suhu dan kelembaban 24/7.
  • Notifikasi Cepat melalui Telegram Bot jika terjadi anomali suhu.
  • Kontrol Jarak Jauh perangkat pendingin (blower) via aplikasi Android.
  • Pencatatan Data historis dalam database lokal untuk analisis performa.

Visualisasi bagaimana data mengalir dari sensor ke pengguna.

Komponen: Perangkat Keras

  • Mikrokontroler ESP8266: Otak dari sistem, terhubung ke sensor dan Wi-Fi.
  • Sensor DHT11: Mengukur suhu dan kelembaban dengan akurasi tinggi.
  • Modul Relay: Menjadi jembatan untuk mengontrol perangkat pendingin.

Komponen: Backend (Server Lokal)

  • Node.js & Express.js: Digunakan untuk membangun server web yang ringan dan cepat.
  • Database SQLite: Menyimpan data historis secara lokal, efisien, dan tanpa konfigurasi rumit.
  • API REST: Menyediakan antarmuka komunikasi antara hardware, aplikasi mobile, dan Telegram bot.

Komponen: Aplikasi Mobile & Notifikasi

  • Aplikasi Android (Kotlin): Dashboard visual untuk pemantauan real-time dan kontrol jarak jauh.
  • Telegram Bot: Mengirimkan notifikasi suhu secara instan dan dapat menerima perintah kontrol.
  • Integrasi Mulus: Aplikasi dan bot terhubung langsung ke server lokal.

Prototipe Fisik

Prototipe fisik sistem ini telah dirancang, dibuat, dan diuji untuk memastikan semua komponen bekerja secara sinergis.

Kendala dan Solusi

  • Kendala: Konfigurasi alamat IP statis untuk server di aplikasi Android. Solusi: Menggunakan alamat IP lokal yang tetap dan memastikan semua perangkat berada di jaringan Wi-Fi yang sama.
  • Kendala: Keterbatasan fitur pada layanan cloud gratis. Solusi: Beralih ke server lokal dengan stack teknologi yang open-source untuk kontrol penuh dan fleksibilitas tanpa biaya.

Hasil Akhir & Manfaat

Sistem berhasil memantau, memberi notifikasi, dan mengontrol perangkat pendingin secara real-time.

Manfaat Utama:

  • Mandiri dan Aman: Tidak bergantung pada layanan pihak ketiga, data tersimpan di server lokal.
  • Fleksibilitas: Memantau dan mengontrol perangkat dari mana saja di dalam jaringan.
  • Efisiensi Biaya: Menggunakan hardware dan software open-source yang terjangkau.
